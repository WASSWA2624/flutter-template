# Project Structure

The template follows `app-rules/project_structure.md`.

```txt
.
|-- analysis_options.yaml
|-- pubspec.yaml
|-- README.md
|-- docs/
|   |-- architecture/
|   |-- decisions/
|   `-- setup/
|-- lib/
|   |-- main.dart
|   |-- bootstrap.dart
|   |-- app/
|   |-- core/
|   |-- features/
|   |-- l10n/
|   `-- shared/
|-- test/
|-- integration_test/
|-- tool/
|-- android/
|-- ios/
|-- web/
|-- windows/
|-- macos/
`-- linux/
```

Feature code is organized under `lib/features/<feature_name>/data`, `domain`,
and `presentation`. Cross-cutting infrastructure belongs in `lib/core`.
Reusable, feature-neutral UI belongs in `lib/shared`. Platform folders are
generated by Flutter and should stay limited to host-specific runner code and
configuration.

See `docs/architecture/app-architecture.md` for dependency direction, layer
boundaries, provider placement, and the starter feature skeleton.
