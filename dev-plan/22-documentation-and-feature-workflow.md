# 22 - Documentation and feature workflow

## Goal
Finalize README, architecture docs, ADRs, and the repeatable feature workflow.

## Applies app rules
- [`documentation_standards.md`](../app-rules/documentation_standards.md)
- [`feature_workflow.md`](../app-rules/feature_workflow.md)
- [`checklists.md`](../app-rules/checklists.md)
- [`coding_conventions.md`](../app-rules/coding_conventions.md)

## Implementation tasks
1. Update README with purpose, platforms, setup, run, test, build, architecture, and folder structure.
2. Create or update architecture docs.
3. Add ADR template or initial ADRs for major choices.
4. Document the new feature workflow.
5. Ensure docs do not contradict app rules.

## Expected output
- Complete README.
- Architecture docs.
- Feature workflow guide.

## Acceptance criteria
- A new developer can set up and run the app using docs.
- Coding agents can follow the feature workflow consistently.
- Docs match actual folders and commands.

## Coding-agent notes
- Do not skip rule references.
- Do not add product-specific behavior unless the step explicitly asks for it.
